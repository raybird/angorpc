# AngoRPC 專案概述

## 專案目標

AngoRPC 旨在建立一個現代化的全端 Web 應用程式，結合以下技術優勢：

- **類型安全**: 透過 TypeScript 和 ORPC 實現端到端類型安全
- **高效能**: 使用 SSR 提升首屏載入速度
- **開發體驗**: 統一的開發環境和工具鏈
- **可維護性**: 清晰的架構分離和代碼組織

## 技術選型理由

### Angular + Universal (SSR)

**選擇理由:**
- 成熟的企業級框架
- 內建 SSR 支援 (Angular Universal)
- 強大的 TypeScript 整合
- 豐富的生態系統

**優勢:**
- SEO 友善
- 首屏載入速度快
- 漸進式水合
- 統一的開發體驗

### ORPC (OpenRPC)

**選擇理由:**
- 類型安全的 RPC 通信
- 自動生成客戶端代碼
- 高效能的二進制協議
- 與 TypeScript 完美整合

**優勢:**
- 端到端類型推導
- 減少 API 文檔維護
- 更好的開發體驗
- 自動化錯誤處理

## 開發原則

1. **類型優先**: 所有 API 和資料結構都先定義 TypeScript 類型
2. **測試驅動**: 重要功能都需有對應的測試
3. **文檔同步**: 代碼變更時同步更新文檔
4. **效能考量**: 在開發過程中持續關注效能指標
5. **安全性**: 實作適當的安全措施和驗證

## 預期效益

- **開發效率**: 類型安全減少運行時錯誤
- **維護性**: 清晰的架構分離
- **效能**: SSR 和 RPC 帶來更好的使用者體驗
- **擴展性**: 模組化設計便於功能擴展
